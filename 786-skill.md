# 你的Skill 是便利贴还是IP？--经验资产化的进化论

---

## 1 一切 AI 的智能，追到底，都是对人类的蒸馏

蒸馏分三层，正好对上 AI 的三个训练阶段。AI获得智能的过程本质上是一层一层的蒸馏人类：预训练阶段蒸馏人类的表达痕迹，后训练阶段蒸馏人类的偏好和价值排序，Agent阶段蒸馏行动路径和任务方法论。Skill就是把一个人的工作方式、思考方式、表达方式压缩成可复制的能力模块。它其实在搭桥——从 AI 最底层的原理，一直铺到工程实践。

AI不是在变得像人，而是人正在被提取、被压榨、被机器复制。工业时代我们把自然资源变成能源，智能时代我们终于玩到了自己身上——把人类的经验、直觉、判断力变成机器智能。

---

## 2 你想保存的东西，在你保存它的那一刻就变形了

你花三天时间把一个专家的经验写成一份 Skill 文档，结构完美，步骤清晰，所有的边界条件都标了。你以为你把知识保存下来了。但实际上，在那个专家做判断的瞬间——他看见一个波形，眉头一皱，说「这个不对」——那一刻发生的不是「执行步骤三」，是一个活着的判断。那个判断里有一万件他没说出来的事：他十年前被这个波形坑过一次，他对这个工具的脾气有直觉，他闻到了某种不对劲但还没法用语言描述。

Skill 的出生就带着一个悖论：你想保存的东西，在你保存它的那一刻就变形了。静止的文字装不住流动的判断力。这解释了三个绕不开的结：

**第一，为什么蒸馏注定丢东西。** 知识不是在档案柜里的，它是在动作里的。从动作里剥离出来的「可保存的部分」，恰好不是让那个人做出判断的东西。

Skill 蒸馏是有损压缩。把经验压成规则，因果链被抹除，信度分配失败。Skill 不是知识，是有损压缩后的碎片。而且你永远不知道你丢了什么——那些被你删掉的细节里，可能刚好藏着下一个任务需要的线索。这不是你蒸馏技术好不好的问题，是信息论层面的天花板。

**第二，为什么进化必须永不停。** 因为每个冻结下来的版本从出生那天就在变旧。你不更新它，它就腐烂。不是写得不好，是写出来的那一刻环境就已经变了。

**第三，为什么循环工程必须存在但永远不够。** 没有循环，你连逼近都做不到；但任何循环都没法捕捉「判断发生的那一个瞬间」。那是活的东西，活的。

## 3 什么真正值钱

走到这里，一个更深的问题浮现了：如果单个 skill 总会过时、总会贬值，那什么才值得投入？

细胞的生命周期很短，但基因会穿越周期。这个比喻很好。你身体里的细胞每几年换一茬，但你的基因一直在。Skill 也一样——单独一个 skill 的生命周期可能就几周，但你用来设计它、测试它、演化它的那套系统，可以用十年。

那我问你一个问题：既然单个 Skill 会过时——蒸馏丢信息、内容会腐烂、碰一下新的任务就可能不管用——那你把工程投入花在「让这个 Skill 更好一点」上，性价比有多低？

单个 Skill 是细胞。它对特定问题生效，跟不上环境变化就会死。但设计、评判、演化和版本控制它的那个 Harness，是基因。基因不干活，但基因决定下一代细胞长什么样。真正的护城河不在 Skill 的仓库里。在设计和演化 Harness 的那个判断力里。


Skill 到 IP 的距离，就是从『跑通一次』到『持续跑通』的距离。

"跑通一次"和"持续跑通"，听起来就差一个字，但中间隔着一整个工程体系。跑通一次，那叫 demo，叫运气，叫恰好那天模型状态好；持续跑通，才叫资产。这中间差的，是你能不能反复验证它、在它坏掉的时候回滚、在它该升级的时候发版。


那"持续跑通"靠什么保证？R2把它收敛成三条，你可以当成一个简易的体检表：

- **结构约束在不在？** ——有没有 Harness 的编排契约，skill 能不能被系统调用、被系统校验？你的 Skill 挂在一个什么样的接口上？它的格式、输入输出、调用契约是不是被系统约束的？还是随便扔在文件夹里的一个 markdown？
    
- **反馈回路在不在？** ——执行结果有没有回流到 skill 的迭代里？还是跑完了就完了？每次这个 Skill 被调用，执行结果有没有被记录下来？成功的条件有没有被提取？失败的模式有没有被归类？这些判断有没有灌回下一次迭代？
    
- **组织背书在不在？** ——有没有人对这个 skill 负责？有没有 DRI？有没有一个明确的人对这玩意负责？有没有版本历史，变更记录，废弃标记？

你的Skill 是便利贴还是IP？--经验资产化的进化论

三条线全有，Skill 就是 IP——活的资产。每用一次，它被验证一次，被更新一点，越来越值钱。缺任何一条，Skill 就是便利贴——今天好用是运气，明天不管用没人知道为什么。

IP 属性不是 Skill 的内容决定的。是系统赋予的。Harness 是三者的容器：没有 Harness，蒸馏是丢信息的、进化是随机的、循环是空转的。管道有了，才有压强；压强有了，才有流动。

一旦这个容器搭起来了，飞轮就开始转：Harness 设计水平 ↑ → AI 可控性 ↑ → 产出质量 ↑ → 度量可信 ↑ → 投入信心 ↑。每一步都是下一步的前置条件，形成五级因果链。系统不只是在完成一次任务，它在让下一次更好。

永远不要问"这个技能应该写什么"。要问"什么样的判断系统站在它背后"。

## 4 没有记忆的循环，只是昂贵的重复

进化是机制，循环工程是把它持续化、自动化的基础设施。但循环工程自己也有个常被忽略的逻辑前提

### 4.1 真循环的分水岭，是"持久记忆—建立关联—自主进化"

循环工程的分水岭不是『能不能让 AI 自己跑』，而是跑完以后，系统有没有记住成功、理解失败、自主进化。

很多人以为循环工程就是"让 AI 自己跑起来"。不是。那只是自动化，不是循环。真正的分水岭在跑完之后：它记不记得自己成功过？懂不懂自己为什么失败？下一次会不会自己改？

### 4.2 失败被重新定价：从代价，变成生产性资产

失败了不再是代价，是写入系统记忆的生产性资产。系统不是随时间衰减，而是随时间变强——试错不再是你在给系统交学费，是系统在给自己长骨头。

它不是在讲技术，是在重定义"失败"的本体。没有记忆的系统，失败就是纯亏；有记忆的系统，失败是喂给骨头生长的钙。"系统在给自己长骨头"——这是一个活着的、在长的东西。

在这个框架下，失败这件事的整个定义都变了。不是你搞砸了需要掩盖的东西。是矿。是写入系统记忆的生产性资产。

传统团队里，一个工程师踩了一个坑，吸收了教训，这个教训长在一个人身上。他跳槽，教训走了。循环工程下的团队，每一次失败都被编码、索引、绑定到具体的触发条件——下一次任何人、任何 AI 触碰到同样条件，系统自动把那条记录拉出来。

### 4.3 工程化的落点，不是 prompt，是学习本身

R2 对循环工程的落点，不是再写一套漂亮的 prompt，而是把学习本身工程化。

这是一条递进的线：先界定什么是真循环，再看真循环怎么重新给失败定价，最后说这个定价要求你把"学习"本身当成工程对象。每一级都在回应上一级留下的"具体怎么办"，不跳，也不绕。

R2 在这个循环上的选择，说穿了就是一件事：不去再写一套更漂亮的 prompt 来优化 Skill。把学习本身工程化。学习是过程，把它变成可操作的对象。

这个飞轮一旦转起来，效果是连锁的。Harness 设计水平上来 → AI 可控性上来 → 产出质量上来 → 度量越来越可信 → 你越来越敢投入 → 投入又进一步优化 Harness……每一跳都可以独立检验、独立优化。而且最后一跳连接到第一跳——闭环了。

## 5 减法，比加法管用

删掉 95% 的 Agent Skills，成功率从 77% 飙到 97%，评估时间从 68 分钟降到 6 分钟。
Skill 的价值与模型能力呈负相关——模型越强，Skill 越不重要。
202 个技能库把通过率砍掉 21%。工具选择准确率从 30 个候选时的 >90% 暴跌到 11,100 个候选时的 13.6%。
这两句一起看，会得出一个让人清醒的结论：你攒 Skill 不是越多越好，是越精越好。候选从 30 涨到 100个，选择准确率从九成跌到一成多。模型越强，它自己越能搞定，你那些多余的反倒成了干扰项。

真正的有用技能读起来像调试日志（具体失败模式 + 具体修复），低质量技能读起来像通用最佳实践讲座。

为什么减法管用？因为"清晰度、完整性、结构"这些表面指标，对实际性能的贡献是负的。越是写得像漂亮最佳实践讲座的，离真实失败模式越远。

## 6 体系不是设计出来的，是跑出来的


框架是脚手架，梯子要扔，一旦你内化了思维习惯，框架就可以被放下。

R2-11 类 Skill 并非学科分类学产物，而是 Harness 编排调用时序与功能角色倒逼出的能力地图。编排先产生调用需求，Skill 类别随后稳定——逆向规划无法复现该全景。

分类是运转倒逼出来的，不是规划出来的。这跟 7 的主旨是一脉的——在运转里长出来的判断，胜过在规划里预设的规则。所以与其纠结"我的 Skill 该分几类"，不如先让它跑起来，跑着跑着，类别自己就长出来了。

不是谁写的 Skill 不行，是系统的分类图跟实际调度需求对不上。有一个 Skill 在编排里被同时用在了「审查」的时序位置和「生成」的时序位置——它到底是哪一类的？

你现在看到的 11 类 Skill，大都是项目先有调用需求，Skill 在响应这些需求的过程中反复被调用、被冲突、被归位，然后类别才慢慢稳定下来。**你永远没法靠坐下来画图复现一个 Skill 体系。** 你必须先让编排系统跑起来，让调用冲突发生，让那些「你以为属于A类其实不是」的错配暴露出来——那些错配，恰好就是真正的分类边界。边界是撞出来的，不是画出来的。

智能体的核心，从来不是『使用 AI』，而是工程治理。过去比拼参数规模，Agent时代比拼谁更擅长把人变成系统。谁能把复杂的工作流程拆成Skill，谁能把一个人的经验压缩成token，谁就能在AI原生组织里获得碾压优势。

真正的知识永远是活的——它在你做判断的那一刻才存在，而那一刻，不需要 Skill。

凡是能说的事情，都能说清楚。不可说的，保持沉默。
